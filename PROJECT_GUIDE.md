# Price Intelligence Tracker — Complete Project Guide

---

## What Is This Project?

**Price Intelligence Tracker** is a backend system that automatically monitors product prices from e-commerce websites.

In simple terms:
- You give it a product URL
- It visits that page using a real browser, reads the price, and saves it to a database
- It does this automatically every 6 hours without you doing anything
- You can query the price history anytime via an API
- If the price drops by more than 5%, it fires an alert

This is exactly the kind of system that companies like Anakin (which collects price and product data from e-commerce sites at scale) build and operate.

---

## The Problem It Solves

Imagine you want to track the price of a product on Amazon. Normally you'd have to:
1. Open the website every day
2. Manually check the price
3. Remember what it was yesterday

This system does all of that automatically. It keeps a full history of every price recorded, lets you query it via an API, and tells you when prices change.

---

## How It Works — Step by Step

### Step 1: You Add a Product URL
You call the API with a product URL:
```
POST /api/products
{"url": "https://books.toscrape.com/catalogue/a-light-in-the-attic_1000/index.html"}
```

### Step 2: The Scraper Runs
The system launches a real invisible browser (Chromium), opens that URL, waits for the page to fully load, and then reads the HTML to extract:
- Product name
- Current price
- Availability
- Star rating

### Step 3: Data is Stored
The scraped data is saved to a PostgreSQL database with a timestamp. Every time the product is scraped again, a new record is added — building up a price history over time.

### Step 4: The API Serves the Data
You can ask the API at any time:
- "What are all my tracked products and their latest prices?"
- "What was the price history for product #1?"
- "Has the price dropped recently?"

### Step 5: The Scheduler Runs Automatically
Every 6 hours, a background job wakes up, visits all tracked product URLs, scrapes fresh prices, and saves new records — without you doing anything.

## Tech Stack — What Each Tool Is and Why It's Used

### Python
The core programming language. Everything is written in Python. It is industry standard for backend systems, scraping, and data pipelines.

### FastAPI
A modern Python web framework used to build the REST API. When you go to `http://localhost:8000/docs`, that entire interactive documentation page is automatically generated by FastAPI. It handles routing (which URL does what), request validation, and response formatting.

**Why FastAPI and not Flask?**
FastAPI is faster, has automatic data validation via Pydantic, generates API docs automatically, and is what production teams use today. Flask is older and simpler but less capable.

### Playwright
A tool from Microsoft that controls a real web browser programmatically. When the scraper runs, it launches an invisible Chromium browser, opens the product URL, waits for JavaScript to fully render the page, and then reads the HTML.

**Why Playwright and not just the `requests` library?**
Many modern websites load their content using JavaScript after the page first loads. A simple HTTP request with `requests` would get the raw HTML before JavaScript runs — meaning the price might not even be in it. Playwright waits for the full page to render, so you always get the real content.

### BeautifulSoup (bs4)
A Python library for parsing HTML. After Playwright gives us the fully rendered HTML of the page, BeautifulSoup is used to navigate and extract specific pieces of it — like the element that contains the price, or the element that contains the product name.

**Workflow:** Playwright renders the page → BeautifulSoup extracts the data.

### SQLAlchemy
A Python library for talking to databases without writing raw SQL. Instead of writing `INSERT INTO products ...`, you write Python code like `db.add(product)` and it handles the SQL for you.

It also defines what the database tables look like through the `models.py` file.

### PostgreSQL (via Neon)
The database where all scraped data is stored. PostgreSQL is a powerful, production-grade relational database used by almost every serious backend system.

The database has two tables:
- **`products`** — stores each tracked product (URL, name, when added)
- **`price_records`** — stores every price snapshot for every product (price, timestamp, availability)

Every time a product is scraped, a new row is added to `price_records`. This is how you build up the price history over time.

**Why Neon?**
Neon provides free cloud-hosted PostgreSQL. No installation required. You get a connection string and use it directly.

### APScheduler
A Python library for running functions automatically on a schedule. In this project, it runs the `scrape_all_products()` function every 6 hours in the background while the API is running.

So the moment you start the API with `uvicorn`, the scheduler also starts silently in the background. You don't have to do anything — it just runs.

### Docker
Docker lets you package the entire application (code + dependencies + environment) into a single container that can run anywhere. The `Dockerfile` contains the instructions for building this container. The `docker-compose.yml` lets you start it with one command: `docker-compose up`.

**Why Docker?**
It means the project works the same on any computer or server. You don't have to worry about "it works on my machine but not yours."

### Anti-Bot Handling (utils.py)
Real websites try to detect and block scrapers. The `utils.py` file handles this by:
- **Rotating user-agents**: Each request pretends to come from a different browser
- **Random delays**: Pauses between requests mimic human browsing speed
- **Realistic headers**: Sends the same HTTP headers a real browser would send

This makes the scraper harder to detect and block.

## The Database Schema

### `products` table
| Column | Type | Description |
|---|---|---|
| id | integer | Unique ID for each product |
| url | text | The product page URL |
| name | text | Product name (from first scrape) |
| created_at | timestamp | When you started tracking it |

### `price_records` table
| Column | Type | Description |
|---|---|---|
| id | integer | Unique ID for each record |
| product_id | integer | Links to the products table |
| price | float | The scraped price |
| availability | text | e.g. "In stock" |
| rating | text | e.g. "Three" (star rating) |
| scraped_at | timestamp | When this price was recorded |

## API Endpoints — What Each One Does

### `POST /api/products`
**What it does:** Adds a new product URL to track. Immediately scrapes it on the first call.

**Request:**
```json
{ "url": "https://books.toscrape.com/catalogue/a-light-in-the-attic_1000/index.html" }
```

**Response:**
```json
{
  "id": 1,
  "name": "A Light in the Attic",
  "url": "https://...",
  "created_at": "2026-04-26T...",
  "latest_price": 51.77
}
```

---

### `GET /api/products`
**What it does:** Lists all tracked products with their latest price.

**Response:**
```json
[
  {
    "id": 1,
    "name": "A Light in the Attic",
    "latest_price": 51.77,
    "created_at": "..."
  }
]
```

---

### `GET /api/products/{id}/history`
**What it does:** Returns full price history for one product — every scrape result ever recorded.

**Response:**
```json
[
  { "price": 51.77, "scraped_at": "2026-04-26T13:15:00" },
  { "price": 49.99, "scraped_at": "2026-04-26T19:15:00" }
]
```

---

### `GET /api/products/{id}/alert`
**What it does:** Compares the last two price records. If the price dropped by more than 5%, it returns `"alert": true`.

**Response (when price dropped):**
```json
{
  "alert": true,
  "drop_percent": 7.5,
  "current_price": 47.99,
  "previous_price": 51.77,
  "message": "Price dropped by 7.5%!"
}
```

---

### `DELETE /api/products/{id}`
**What it does:** Removes a product from tracking and deletes all its price history from the database.

---

### `GET /`
**What it does:** Health check. Confirms the API is alive and running.

## How to Run the Project (Setup Instructions)

### Requirements
- Python 3.10 or higher
- Git
- A free Neon account (https://neon.tech)

### Step 1 — Clone the Repository
```bash
git clone https://github.com/YOUR_USERNAME/price-tracker.git
cd price-tracker
```

### Step 2 — Create Virtual Environment
```bash
python -m venv venv
venv\Scripts\activate      # Windows
```

### Step 3 — Install Dependencies
```bash
pip install -r requirements.txt
playwright install chromium
```

### Step 4 — Configure the Database
Create a `.env` file in the root folder:
```
DATABASE_URL=postgresql://your_user:your_password@your_host/neondb?sslmode=require
```
Get your connection string from https://console.neon.tech.

### Step 5 — Run the API
```bash
uvicorn api.main:app --reload
```

### Step 6 — Access the API
Open your browser and go to:
```
http://localhost:8000/docs
```

### Step 7 — Add Your First Product
In the Swagger UI, use `POST /api/products` with:
```json
{ "url": "https://books.toscrape.com/catalogue/a-light-in-the-attic_1000/index.html" }
```

### Step 8 — Run with Docker (Optional)
```bash
docker-compose up --build
```

## What Happens Automatically (Without You Doing Anything)
Once the API is running, the scheduler runs `scrape_all_products()` every 6 hours in the background. It iterates over every product in the database, scrapes a fresh price, and saves a new record. This builds up the price history automatically over time.
